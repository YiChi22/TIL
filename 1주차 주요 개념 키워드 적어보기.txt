1. 버전 관리
	누가, 언제, 어떤 내용을 변경했는지 가 적혀있어야 하며 협업이나 프로젝트 관리에 중요
2. Git 초기화
	컴퓨터에 있는 폴더를 Git이 관리하는 프로그램으로 만들어줌다.
3. add, commit
	추가 혹은 수정된 파일을 add하고 commit 한다. add 할 때는 연관 파일로 묶어서 나눠 해주어야 나중에 수정사항이 있을 때 확인이 수월하다
4. 원격 repo, 로컬 repo
	로컬은 내 컴퓨터, 원격은 GitHub 같은 협업할 때 수많은 로컬들이 붙어서 관리
5. 히스토리
	작업 내역과 변경 코드 같은 것들이 전부 저장된다.
6. push, pull
	push는 내 컴퓨터에서 원격 repo로 보내는 것 pull은 원격에서 내 컴퓨터로 가져오는 것
7, tracking, clone
	tracking은 로컬 repo와 원격 repo 를 연결해주는 것이며 clone은 로컬 repo가 없을 때 원격 repo를 복사해오는 것이다.
